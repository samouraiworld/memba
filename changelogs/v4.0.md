## v4.0.0 (2026-04-08) — GnoBuilders Quest System, Security Hardening & GovDAO Compatibility

### Added

- **GnoBuilders — Gamified Onboarding** (#274)
  - 85 quests across 4 categories (Developer, Everyone, Champion, Hidden)
  - 8-tier rank system: Newcomer → Gno Guardian (2,000 XP)
  - On-chain + off-chain + social + self-report verification types
  - Prerequisite chains, difficulty levels, seasonal quests
  - Quest Hub browser with search, category tabs, difficulty filters
  - Quest Detail page with prerequisite chain visualization
  - Leaderboard with XP rankings
  - Badge NFT minting infrastructure (GRC721)
  - Backend: quest completion RPC, admin review endpoints, XP validation
  - Database migration: `010_gnobuilders.sql`
  - 1,588 tests across 72 files

- **Quest Completion Celebration** (#276)
  - XP overlay with fade-in + pop animation on quest verification
  - Auto-dismiss after 4 seconds

- **IPFS Gateway Fallback** (#275)
  - 3-gateway chain: Lighthouse → ipfs.io → dweb.link
  - `fetchFromIpfs()` with automatic retry, `handleAvatarError()` for `<img>` fallback

- **GovDAO ExecuteOrRejectProposal** (#275)
  - Support for gno#5261: graceful proposal rejection on execution error
  - `GOVDAO_EXECUTE_FUNC` configurable constant for upstream migration

- **Retry Buttons on Error States** (#275)
  - `onRetry` wired to ErrorToast in Dashboard, DAOHome, DAOList, ProposalView, MultisigHub, Treasury

- **Marketplace Skeleton Loaders** (#275)
  - SkeletonCard grid during agent loading

### Fixed

- **Proposal ID 0 rejected by AI analysis** (#275)
  - Gno proposals are 0-indexed; validation changed from `<= 0` to `< 0`
  - User-reported bug on `/test11/dao/gno.land/r/gov/dao/proposal/0`

- **502 Bad Gateway on AI consensus analysis** (#275)
  - WriteTimeout increased from 30s to 90s (10-model fan-out needs ~41s)
  - Added 55s handler-level context timeout
  - Response write errors now logged instead of silently discarded

- **Render output markdown escaping** (#275)
  - `unescapeMarkdown()` strips backslash-escaped chars from gno#5418
  - Applied to proposal titles and descriptions in both list and detail parsers

- **parseInt radix** (#275)
  - Added radix 10 to parseInt calls in WizardStepConfig, WizardStepMembers, Settings, Marketplace

- **TreasuryProposal placeholder** (#275)
  - Improved from "g1recipient..." to "g1abc...xyz (Gno address)"

### Infrastructure

- CI: Go 1.25 → 1.25.9 (fixes 4 stdlib vulnerability scan failures)
- 9 new backend tests, 8 new frontend tests
